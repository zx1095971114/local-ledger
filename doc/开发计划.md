# 本地账本桌面应用开发计划

## 一、项目概述

### 1.1 项目目标
开发一个跨平台桌面应用，用于管理个人账单数据，支持从一木记账导入数据，多维度查看账单，手动添加账单，并与本地Ollama模型集成获取理财建议。

### 1.2 核心功能
- ✅ 导入一木记账账单（XLS格式，支持覆盖导入）
- ✅ 多维度查看账单（时间、类别、账户等）
- ✅ 手动添加/编辑/删除账单
- ✅ **账户管理**：独立页面维护账户主数据（增删改查）；从账户穿透至账单列表并自动带上该账户筛选条件
- ✅ 与本地Ollama模型集成，获取理财建议
- ✅ 数据本地存储（SQLite数据库）

### 1.3 学习目标
- 🎯 AI调用：学习如何调用本地Ollama模型API
- 🎯 前端UI：使用Vue3构建现代化界面
- 🎯 跨平台应用：使用Electron构建桌面应用

## 二、技术选型

### 2.1 桌面应用框架：Electron + Vue3
**选择理由：**
- Electron成熟稳定，生态丰富
- Vue3是您熟悉的框架，学习成本低
- 跨平台支持（Windows、macOS、Linux）
- 可以使用Node.js作为后端，也可以调用Python脚本

**技术栈：**
- **主开发语言**: TypeScript（主进程和渲染进程都使用TS）
- **前端框架**: Vue 3 + TypeScript
- **UI组件库**: Element Plus（已选择，您熟悉的组件库）
- **状态管理**: Pinia
- **路由**: Vue Router（已配置多页面路由）
- **构建工具**: Vite
- **桌面框架**: Electron

**TypeScript使用说明：**
- ✅ **主进程（main）**: 使用TypeScript编写，编译为JavaScript运行
- ✅ **渲染进程（renderer）**: Vue3组件使用TypeScript
- ✅ **预加载脚本（preload）**: 可以使用TypeScript
- ✅ **类型安全**: 全程TypeScript，提供更好的代码提示和错误检查

### 2.2 数据存储：SQLite
**选择理由：**
- 轻量级，无需单独安装数据库服务
- 适合个人使用场景
- Node.js有成熟的SQLite库（better-sqlite3）

**SQLite库选择：**
- **推荐库**: `better-sqlite3`（同步API，性能好）
- **TypeScript支持**: ✅ 有完整的TypeScript类型定义（@types/better-sqlite3）
- **使用方式**: 在Electron主进程中用TypeScript直接调用
- **备选库**: `sql.js`（纯JavaScript实现，但性能略差）

**示例代码：**
```typescript
// main/database/db.ts
import Database from 'better-sqlite3';

const db = new Database('ledger.db');

// TypeScript完全支持，有类型提示
db.exec(`
  CREATE TABLE IF NOT EXISTS bills (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL,
    ...
  )
`);
```

### 2.3 AI集成：Python + Ollama SDK
**选择理由：**
- Ollama官方提供Python SDK，文档完善
- Python脚本易于调试和维护
- 通过Electron的child_process调用Python脚本
- 后续可以学习如何优化prompt和模型参数

**备选方案：**
- 如果Ollama提供Node.js SDK，也可以直接使用Node.js调用

### 2.4 数据处理：Node.js + Python
- **XLS解析**: 使用Node.js的`xlsx`库或Python的`pandas`（复用现有工具）
- **数据验证**: Node.js端处理
- **数据导入**: Node.js调用Python脚本或直接使用Node.js库

**数据库表结构示例：**
```sql
-- 账单表
CREATE TABLE bills (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL,              -- 日期
    type TEXT NOT NULL,              -- 收支类型：收入/支出/转账（CHECK 与迁移见下）
    amount REAL NOT NULL,            -- 金额
    category TEXT,                   -- 类别（转账时为空）
    subcategory TEXT,                -- 子类（转账时为空）
    account TEXT,                    -- 账户（支出/收入；转账见 account_from/account_to）
    account_from TEXT,               -- 转出账户（仅 type=转账）
    account_to TEXT,                 -- 转入账户（仅 type=转账）
    ledger TEXT,                     -- 账本
    reimbursement_account TEXT,      -- 报销账户
    reimbursement_amount REAL,       -- 报销金额
    refund_amount REAL,              -- 退款金额
    note TEXT,                       -- 备注
    tags TEXT,                       -- 标签（JSON数组）
    address TEXT,                    -- 地址
    created_user TEXT,               -- 创建用户
    discount REAL,                   -- 优惠
    other TEXT,                      -- 其他
    attachments TEXT,                -- 附件（JSON数组）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_bills_date ON bills(date);
CREATE INDEX idx_bills_type ON bills(type);
CREATE INDEX idx_bills_category ON bills(category);
CREATE INDEX idx_bills_account ON bills(account);

-- 说明：已有库需迁移放宽 type 的 CHECK 并 ALTER 增加 account_from/account_to；详见 [add-bill-spec.md](./add-bill-spec.md) 第 4.2 节与 `electron-app/electron/main/database/db.ts` 迁移逻辑。

-- 账户主数据表（供账户管理页维护；与 bills.account 对齐）
-- 关联策略（二选一，实现时择一并在迁移中统一）：
--   A. bills.account 存账户「名称」字符串，与 accounts.name 一致（导入时按名称匹配或自动补全账户）
--   B. 后续迁移为 bills.account_id 外键关联 accounts.id（名称可改、历史账单仍指向 id）
CREATE TABLE IF NOT EXISTS accounts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,           -- 账户名称（与一木「账户」列对应）
    type TEXT,                           -- 类型：现金/储蓄卡/信用卡等（可选）
    currency TEXT DEFAULT 'CNY',         -- 币种（可选）
    note TEXT,                           -- 备注
    sort_order INTEGER DEFAULT 0,      -- 列表排序
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_accounts_name ON accounts(name);
```

**账户与账单一致性（实现要点）：**
- 账户管理页的「查」需支持按关键字过滤名称；「删」前需定义策略：禁止删除仍有账单引用的账户 / 或仅允许删除无关联数据的账户，并给用户明确提示。
- 账单表单中「账户」下拉建议来源于 `accounts` 表，减少自由文本与主数据不一致。

**学习重点：**
- SQLite数据库操作（使用better-sqlite3）
- TypeScript类型定义（为数据库操作定义接口类型）
- SQL查询优化
- 数据库迁移管理

**技术细节：**
- `better-sqlite3`是同步API，性能好，适合Electron主进程
- 需要安装：`npm install better-sqlite3` 和 `npm install -D @types/better-sqlite3`
- 数据库操作封装为TypeScript类/模块，提供类型安全的接口

## 五、技术难点与解决方案

### 5.1 Electron主进程与渲染进程通信
**难点：** 安全地实现进程间通信
**解决方案：** 
- 使用Context Isolation + Preload脚本
- 通过`ipcMain`和`ipcRenderer`进行通信
- 避免直接暴露Node.js API到渲染进程

### 5.2 大数据量处理
**难点：** 账单数据量大时，列表渲染可能卡顿
**解决方案：**
- 使用虚拟滚动（vue-virtual-scroller）
- 分页加载
- 数据库索引优化

### 5.3 Python脚本调用
**难点：** Electron调用Python脚本的稳定性
**解决方案：**
- 使用child_process.spawn
- 实现超时机制
- 完善的错误处理
- 考虑将Python脚本打包进应用（PyInstaller）

### 5.4 AI响应时间
**难点：** Ollama模型响应可能较慢
**解决方案：**
- 显示加载状态
- 支持取消请求
- 考虑流式响应（如果Ollama支持）
- 缓存常见问题的回答

### 6.4 项目中的实际应用示例

#### **示例1：数据库操作（主进程）**

```typescript
// main/database/db.ts
import Database from 'better-sqlite3';

// 定义Bill接口（类似Java的实体类）
interface Bill {
    id?: number;  // 可选，因为插入时可能没有
    date: string;
    type: '收入' | '支出';  // 联合类型，只能是这两个值之一
    amount: number;
    category?: string;
    note?: string;
}

// 数据库服务类（类似Java的Service层）
class BillDatabase {
    private db: Database.Database;
    
    constructor(dbPath: string) {
        this.db = new Database(dbPath);
        this.initTables();
    }
    
    // 初始化表（类似Java的DDL）
    private initTables(): void {
        this.db.exec(`
            CREATE TABLE IF NOT EXISTS bills (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                date TEXT NOT NULL,
                type TEXT NOT NULL,
                amount REAL NOT NULL,
                category TEXT,
                note TEXT
            )
        `);
    }
    
    // 插入账单（返回插入的ID）
    insert(bill: Bill): number {
        const stmt = this.db.prepare(`
            INSERT INTO bills (date, type, amount, category, note)
            VALUES (?, ?, ?, ?, ?)
        `);
        const result = stmt.run(
            bill.date,
            bill.type,
            bill.amount,
            bill.category || null,
            bill.note || null
        );
        return result.lastInsertRowid as number;
    }
    
    // 查询所有账单
    findAll(): Bill[] {
        const stmt = this.db.prepare('SELECT * FROM bills ORDER BY date DESC');
        return stmt.all() as Bill[];
    }
    
    // 根据ID查询
    findById(id: number): Bill | null {
        const stmt = this.db.prepare('SELECT * FROM bills WHERE id = ?');
        const bill = stmt.get(id) as Bill | undefined;
        return bill || null;  // 如果没找到返回null
    }
    
    // 删除账单
    delete(id: number): boolean {
        const stmt = this.db.prepare('DELETE FROM bills WHERE id = ?');
        const result = stmt.run(id);
        return result.changes > 0;  // 返回是否删除成功
    }
}

export { BillDatabase, Bill };
```

import { defineStore } from 'pinia';
import type { Bill, BillQuery } from '@/types/bill';

export const useBillStore = defineStore('bill', {

        error: null as string | null,

        },

            return state.bills.filter(bill => bill.type === '收入');

            return state.bills.filter(bill => bill.type === '支出');
        },

            this.loading = true;
            this.error = null;
            try {
                this.bills = await window.electronAPI.getBills(query);
            } catch (error) {
                this.error = error instanceof Error ? error.message : '未知错误';

                this.loading = false;

            this.bills.unshift(newBill);  // 添加到数组开头

### 6.5 常见错误和解决方案

#### **错误1：类型不匹配**

```typescript
// ❌ 错误
let id: number = "123";  // 类型错误：string不能赋值给number

// ✅ 正确
let id: number = 123;
// 或
let id: string | number = "123";  // 使用联合类型
```

#### **错误2：可能为null/undefined**

```typescript
// ❌ 错误
function getBill(id: number): Bill {
    const bill = findBill(id);
    return bill;  // 错误：bill可能是null
}

// ✅ 正确
function getBill(id: number): Bill | null {
    return findBill(id);  // 明确返回类型可能为null
}

// 使用时检查
const bill = getBill(1);
if (bill) {  // TypeScript知道这里bill不是null
    console.log(bill.amount);
}
```

#### **错误3：访问可能不存在的属性**

```typescript
// ❌ 错误
const bill: Bill = { id: 1, date: "2024-01-01", amount: 100 };
console.log(bill.note.toUpperCase());  // 错误：note可能是undefined

// ✅ 正确
if (bill.note) {
    console.log(bill.note.toUpperCase());
}
// 或使用可选链
console.log(bill.note?.toUpperCase());  // 如果note是undefined，返回undefined
```

## 十、注意事项

1. **数据安全**
   - 账单数据涉及隐私，确保数据本地存储
   - 如果后续需要云端功能，注意数据加密

2. **性能考虑**
   - 数据库索引要合理设计
   - 大数据量时考虑分页或虚拟滚动

3. **用户体验**
   - 操作要有明确的反馈
   - 错误提示要友好
   - 支持撤销操作（重要操作）

4. **代码质量**
   - 虽然个人项目，但建议保持代码规范
   - 关键功能要有注释
   - 考虑单元测试（可选）

## 十一、开始开发

### 第一步：环境准备
1. 安装Node.js（推荐18+版本）
2. 安装Python（推荐3.9+版本）
3. 安装Ollama并下载模型（如llama3、qwen等）
4. 安装Git

### 第二步：创建项目
```bash
# 使用electron-vite创建项目（推荐，自动配置TypeScript）
npm create electron-vite@latest electron-app -- --template vue-ts

# 进入项目目录
cd electron-app

# 安装SQLite数据库库
npm install better-sqlite3
npm install -D @types/better-sqlite3

# 安装其他依赖（electron-vite模板已包含基础依赖）
npm install

# 或手动搭建（不推荐，配置复杂）
mkdir electron-app
cd electron-app
npm init -y
npm install electron electron-builder --save-dev
npm install vue@next pinia vue-router --save
npm install better-sqlite3 --save
npm install typescript @types/node --save-dev
npm install -D @types/better-sqlite3
```

### 第三步：开始阶段一开发
按照阶段一的任务清单逐步实现。
